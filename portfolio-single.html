Sterling Pro: Transforming Analytics for Better Insights

Overview

The Sterling Pro Admin project focused on redesigning the Analytics section to improve user experience and accessibility of critical data. As the lead UX designer, I aimed to address usability challenges and enhance data presentation for better decision-making. This project was vital because insights derived from analytics directly impact operational efficiency and strategic planning. Through thoughtful redesign, we sought to create a more intuitive, comprehensive overview of platform activities that empowers users.

Problem

The existing analytics section suffered from poor usability, scattered data presentation, and limited actionable insights.

Solution

We implemented a streamlined dashboard design that enhances usability and prioritizes critical metrics for better visibility.

Client

Sterling Bank

Year

2020-present

Category

Fintech Dashboard

Goals

The redesign aimed to deliver an intuitive and actionable overview of the platform's activities, improving data clarity and decision-making speed.

Goal 1

Provide a clear dashboard overview of onboarding statuses.

Goal 2

Enable real-time tracking of transaction performance.

Goal 3

Implement comparative analysis for user activity and complaints.

Interview

I conducted interviews with 20 stakeholders and users to identify pain points within the analytics section. Participants were chosen for their familiarity with the platform, ensuring diverse feedback on usability challenges. The goal was to understand their experiences and expectations for actionable insights. This feedback guided our redesign strategy.

UI Design

The final UI decisions were centered around creating a visually appealing and functional dashboard that aligns with the brand identity. We utilized a distinct color palette to improve data visualization and ensure clarity across various metrics. By opting for a stacked bar chart and toggle mechanisms, we addressed the initial problems of clutter and redundancy, allowing users to engage with the data effectively and derive actionable insights.

Usability Testing

Usability testing involved a group of 15 participants representing our target users. We observed their interactions with the redesigned dashboard, focusing on navigation ease and data comprehension. Key findings revealed areas for further refinement, emphasizing the need for intuitive interactions and clear data representation.

Problem 1

The dashboard’s reactive, ad-hoc design resulted in redundant data, a poor information hierarchy, wasted space, and limited scalability, making it more difficult for users to access and act on critical insights quickly.

Solution 1

The layout was streamlined by consolidating widgets, reducing clutter, and freeing space for high-value analytics like transfer type breakdowns and feature usage insights, improving visibility, decision-making speed, and scalability.

Problem 2

The existing Transaction section lacked detailed insights into which channels businesses used most, limiting the ability to track trends and optimize operations.

Solution 2

A Transfer Channel Breakdown was added to categorize transaction volumes and values by channel, giving stakeholders clearer visibility for monitoring, decision-making, and improvements.

Problem 3

The Feature Usage section’s uniform colors and cluttered bar charts made it hard to compare feature performance, while the Complaint Analysis section’s separate views created visual overload and wasted space.

Solution 3

Unique colors and a stacked bar chart improved feature usage clarity, while a dropdown toggle replaced separate complaint views, reducing clutter and making both sections more readable, navigable, and decision-friendly.

Learnings

  • Understanding user needs is crucial for effective design.
  • Iterative testing can significantly improve usability and clarity.
  • Clear visual hierarchy enhances user comprehension and decision-making.

Next Steps

  • Conduct follow-up user testing to refine the design further.
  • Gather feedback post-launch to identify areas for ongoing improvement.
  • Explore additional analytics features for future iterations.
